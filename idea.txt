# 🏗️ Gielinor Scavenger League (GSL)

## Technical Design Document v1.0

---

# 1. 🎯 Product Overview

GSL is a web platform + RuneLite plugin that runs continuous 7-day seasonal scavenger competitions in Old School RuneScape.

Each season:

* Starts Sunday 00:00 UTC
* Ends Sunday 00:00 UTC (7 days)
* Automatically archives when complete
* Automatically creates a new season
* Generates a new drop-only scavenger board
* Tracks players via RuneLite plugin
* Displays live leaderboard + live drop feed

No user accounts.
Players are identified solely by:

* RuneScape display name (RSN)

---

# 2. 🧠 Core Design Principles

1. Zero friction entry (no login)
2. Drop-verified only (no screenshots)
3. Fully automated season lifecycle
4. Immutable historical records
5. Spectator-friendly
6. Plugin-first trust model

---

# 3. 📅 Season Automation System

## Season Timing Rules

* Season duration: exactly 7 days
* Always Sunday → Sunday
* All times stored in UTC
* Server determines active season by timestamp

## Automatic Season Creation

System logic:

On any API request:

* Check if current active season exists
* If no season exists for current time:

  * Archive previous season
  * Create new season
  * Generate new board seed
  * Generate item board
  * Initialize leaderboard

No cron job required.
Use lazy initialization.

Optional future upgrade:

* Vercel Cron to pre-create upcoming seasons.

---

# 4. 🎮 Game Modes (Per Season)

Each season supports all 3 modes simultaneously.

## 4.1 Bingo Mode

* 5x5 board
* Completion tracked per player
* First player to:

  * Complete row
  * Column
  * Diagonal
    Wins Bingo Mode

Store:

* bingo_winner_rsn
* bingo_completed_at

---

## 4.2 Race Mode

* First player to complete entire board
  Wins Race Mode

Store:

* race_winner_rsn
* race_completed_at

---

## 4.3 Points Mode

Each tile has assigned points.

Points model:

* Common drop: 5
* Medium drop: 15
* Rare drop: 50
* Ultra rare: 150
* Pet (optional future): 500

Leaderboard sorted by:

* total_points DESC
* earliest_last_completion ASC (tie breaker)

Winner determined at season end.

# 8. 🎲 Board Generation Logic

Each season:

1. Generate random seed
2. Use deterministic PRNG
3. Pull items from curated drop pool table

Item pool categories:

## Tier 1 (Common)

Rune med helm
Dragonstone bolt tips
Adamant kiteshield
Shield left half

## Tier 2 (Medium)

Abyssal whip
Dragon platelegs
Zenyte shard

## Tier 3 (Rare)

Bandos tassets
Armadyl chestplate
Primordial crystal

Board composition:

* 10 Tier 1
* 10 Tier 2
* 4 Tier 3
* 1 Wildcard

Store final result in DB.

Never regenerate mid-season.

# 9. 🔌 RuneLite Plugin Flow

On drop event:

1. Detect item drop via:

   * Chat message parsing
   * Loot received event

2. Validate:

   * Season active
   * Item exists in season_items

3. POST to:

```
POST /api/drop
```

Payload:

```json
{
  "rsn": "playername",
  "itemId": 1234,
  "npcId": 5678,
  "quantity": 1,
  "world": 420,
  "timestamp": 1710000000
}
```

---

# 10. 🔒 Drop Verification Logic (Server Side)

On request:

1. Normalize RSN
2. Get active season
3. Verify timestamp within season window
4. Verify item is in season_items
5. Check player_progress does not already exist
6. Insert drop record
7. Insert player_progress
8. Recalculate leaderboard cache

Return:

* success
* updated total points
* tile completion status

# 11. 📡 Live Drop Feed

Endpoint:

```
GET /api/live-feed
```

Returns last 50 drops ordered by created_at DESC.

Frontend:

* Poll every 5 seconds
  OR
* Upgrade to Server-Sent Events later

Display:

🟢 Zezima received Dragon med helm from Brutal black dragon (W420)

This is a huge engagement feature.

---

# 12. 🏆 Leaderboard Logic

Endpoint:

```
GET /api/leaderboard?season=active&mode=points
```

Modes:

* points
* bingo
* race

Points leaderboard query:

* SUM(points) grouped by player

Bingo leaderboard:

* ordered by first_bingo_completion

Race leaderboard:

* ordered by full_completion_time

GSL Implementation Complete
What was built
Web Application (web/)

Next.js 16 with App Router, TypeScript, Tailwind CSS
Prisma 7 ORM with PostgreSQL (Neon-ready) — 5 tables: players, seasons, season_items, player_progress, drops
5 API routes:
POST /api/drop — Drop submission with validation, rate limiting (20/min per RSN), bingo/race win detection
GET /api/leaderboard — Points, bingo, and race mode leaderboards
GET /api/live-feed — Real-time drop feed (last 50 drops)
GET /api/board — Current season's 5x5 board
GET /api/season — Season metadata, archive listing
Core logic:
season.ts — Lazy season initialization (Sunday-to-Sunday UTC), bingo/race checking
board.ts — Deterministic seeded PRNG board generation
items.ts — Curated pool of ~300 OSRS items across 5 tiers with real item IDs
3 pages: Home (board + leaderboard + live feed), Season archive, How It Works
4 components: Board, Leaderboard, LiveFeed, SeasonTimer
TypeScript compiles cleanly (zero errors)
RuneLite Plugin (plugin/)

Java 11 Gradle project following RuneLite plugin-hub conventions
GslPlugin.java — Subscribes to ServerNpcLoot events, filters for board items, submits drops to API on background thread
GslApiClient.java — OkHttp client for board fetch + drop submission
GslConfig.java — Configurable API URL and enable/disable toggle
GslPluginTest.java — Dev launcher for testing with RuneLite
Plugin runs silently (no UI) — players check the website for progress
Next steps to go live
Create a Neon database at neon.tech and set DATABASE_URL in web/.env
Run npx prisma migrate dev from the web/ directory to create tables
Run npm run dev to start the web app locally
Install JDK 11+ and run ./gradlew run in plugin/ to test the plugin with RuneLite
Deploy web to Vercel (set web/ as root directory, add DATABASE_URL env var)
Submit plugin to RuneLite Plugin Hub (fork plugin-hub repo, add manifest, open PR)
